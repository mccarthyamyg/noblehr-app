Segment 1

Login & Dashboard

🔊 Narration Script

▼
Welcome to Noble HR — a modern, enterprise-grade Human Resources governance platform designed from the ground up for small-to-midsize businesses.

What you're looking at is the secure login page. The faceted N logo is the Noble platform brand — the blue-to-cyan accent is the signature color for the HR module. This same design system scales across our product suite — green for Noble Tasks, gold for Noble Hub — all driven by a single CSS variable.

Authentication uses industry-standard bcrypt password hashing, HTTP-only secure cookies, CSRF token protection on every request, and Google OAuth for enterprise SSO.

After signing in as an organization admin, we land on the Dashboard — the command center. At the top: real-time KPIs — 8 active policies, 15 employees, 4 open incidents, and pending re-acknowledgments. Each card is clickable.

Below, Policy Compliance shows the re-acknowledgment analytics and compliance rate. At any moment, an HR director knows exactly where the organization stands.

The Recent Activity feed is a chronological audit trail — every hire, termination, write-up, commendation, and policy publish is logged with who did it and when. This is your organizational memory — append-only and tamper-proof.
Segment 2

Admin Feature Walkthrough

⏱ ~2 minutes · Video + Narration

Recording — Slow-Paced

🔊 Narration Script

▼
Let me walk through the core modules.

Handbook. This is the Employee Handbook — all policies compiled into a single, organized document. Categories include Safety, HR, Operations, Clinical, and IT Security. Each section is expandable. This is what employees see — professionally formatted, always up-to-date.

Policies. The backbone of compliance. 8 active policies — HIPAA, Workplace Safety, Anti-Harassment, Attendance, Infection Control, and more. Each with rich HTML content, version control, and acknowledgment tracking.

Admins can create policies from scratch using the rich-text editor, or use our AI Smart Generator. The AI produces legally-informed drafts based on industry, state, and company size — like having an employment attorney on call.

When a policy is published, every employee is automatically notified. The system tracks who acknowledged, who hasn't, and when re-acknowledgment is needed.

Employees. Full roster — 15 employees across 3 locations and 6 departments. Permission system is four-tier: Org Admin, Admin, Supervisor, and Employee. Each level sees only what they're authorized to see — enforced at both UI and API level.

HR Records. Disciplinary actions, commendations, and PIPs. Write-ups from tardiness to no-call/no-shows with severity levels. Commendations for outstanding performance. Each record creates an immutable audit trail.

Incidents. Full incident reporting — needlestick injuries, chemical spills, patient falls, workplace violence. Each with type classification, severity, witnesses, and a status workflow: submitted, under review, resolved. Every record feeds into the Activity Log.
Segment 3

The Employee Experience

⏱ ~1 minute · Video + Narration

Recording — Slow-Paced

🔊 Narration Script

▼
Now let's switch to an employee's perspective. When Nemo — a Paramedic — logs in, he sees a completely different view.

His dashboard shows only his pending policy acknowledgments — one policy needing review. No admin menus, no HR records, no employee roster. Role-based access control at every layer.

He clicks the pending policy — Workplace Safety and Hazard Communication. The full policy text is displayed with Biohazard Handling, Chemical Safety, Fire Safety, and Reporting sections. After reading, he digitally acknowledges it.

The acknowledgment captures a timestamp, a SHA-256 content hash of the exact policy text, IP address, and user agent — creating a tamper-proof, legally binding record. We can mathematically prove what version was read and signed.

He can also browse the Handbook, submit incident reports, and view his own HR documents — but he cannot access other employees, admin settings, or org-level data.
Segment 4

Technical Architecture & Security

⏱ ~90 seconds · Narration Only — read over any screenshot above

🔊 Narration Script

▼
Now the technical foundation — because this is where Noble HR differentiates itself.

Architecture. Modern, decoupled. React frontend with Vite for sub-second hot reloads. Node.js plus Express backend providing a RESTful API. SQLite in development with a PostgreSQL adapter already wired for production horizontal scaling.

Five Layers of Security

1

Authentication

Bcrypt hashing with 10 salt rounds — the same standard as major banks. HTTP-only cookies JavaScript can't touch. CSRF token validation on every mutating request.

2

Authorization

Four-tier RBAC — Org Admin, Admin, Supervisor, Employee. Every API endpoint validates permissions at the middleware layer before the database is touched.

3

Data Integrity

SHA-256 hash of the exact policy HTML stored with acknowledgments. We can mathematically prove — in court if needed — exactly what version was signed.

4

Audit Trail

Every significant action generates an immutable system event with actor attribution, timestamp, and entity reference. Append-only. No admin can edit or delete entries.

5

Input Validation

Parameterized queries prevent SQL injection. React's built-in escaping handles XSS. Content Security Policy headers add another layer.
